# Quick Start

¡Bienvenido al sistema de Ecosistemas! Esta guía rápida te llevará a través de los pasos necesarios para configurar y probar el entorno del agente, optimizado para procesos de soporte técnico.

## Paso 1: Configurar Visual Studio Code

### Instalar Visual Studio Code

Descarga e instala Visual Studio Code desde [aquí](https://code.visualstudio.com/).

### Configurar el `launch.json`

Configura el depurador agregando el archivo `launch.json` en la carpeta `.vscode` de tu proyecto.

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Python Debugger: FastAPI",
            "type": "debugpy",
            "request": "launch",
            "module": "uvicorn",
            "args": [
                "main:app",
                "--host",
                "0.0.0.0",
                "--port",
                "8080"
            ],
            "jinja": true
        }
    ]
}
```

## Paso 2: Configurar una Base de Datos MongoDB

Configura una base de datos MongoDB para almacenar datos del sistema.

### Crear una Base de Datos MongoDB

Puedes usar MongoDB Atlas para crear una base de datos en la nube:
1. Regístrate en [MongoDB Atlas](https://www.mongodb.com/cloud/atlas).
2. Crea un clúster y una base de datos.
3. Obtén la cadena de conexión para conectarte a tu base de datos.

## Paso 3: Configurar Variables de Entorno

Crea un archivo `.env` en la raíz de tu proyecto y agrega las siguientes variables de entorno. Estas configuraciones son necesarias para conectar a servicios externos y bases de datos.

```ini
# MONGO DB
MONGO_CONNECTION_STRING=<your-mongodb-connection-string>

# COGNITIVE SERVICES
OPENAI_API_KEY=<your-openai-api-key>

# MESSAGING
TWILIO_ACCOUNT_SID=<your-twilio-account-sid>
TWILIO_AUTH_TOKEN=<your-twilio-auth-token>

# SERVICE DESK
JIRA_SERVER=https://<your-jira-server>/
JIRA_API_KEY=<your-jira-api-key>
JIRA_PROJECT=<your-jira-project>
JIRA_USER=<your-jira-user-email>
JIRA_DELEGATION_USER=<your-jira-delegation-user-email>
JIRA_DONE_STATUS_CODE=<your-jira-done-status-code>
```

### Descripción de las Variables de Entorno

- **MONGO_CONNECTION_STRING**: La cadena de conexión para tu base de datos MongoDB.
- **OPENAI_API_KEY**: Tu clave API para el servicio OpenAI.
- **TWILIO_ACCOUNT_SID**: El SID de tu cuenta Twilio utilizada para servicios de mensajería.
- **TWILIO_AUTH_TOKEN**: El token de autentificación para tu cuenta Twilio.
- **JIRA_SERVER**: La URL de tu servidor JIRA.
- **JIRA_API_KEY**: La clave API para acceder a tu cuenta JIRA.
- **JIRA_PROJECT**: El código del proyecto JIRA.
- **JIRA_USER**: El email del usuario utilizado para autentificación en JIRA.
- **JIRA_DELEGATION_USER**: El email del usuario delegado en JIRA.
- **JIRA_DONE_STATUS_CODE**: El código de estado para indicar que una tarea está completada en JIRA.

## Paso 4: Configurar el Entorno Virtual y Instalar Dependencias

### Crear y Activar un Entorno Virtual

Es recomendable utilizar un entorno virtual para gestionar las dependencias de tu proyecto. 

```bash
# Crear un entorno virtual con Python
python -m venv venv

# Activar el entorno virtual
# En Windows:
venv\Scripts\activate

# En macOS/Linux:
source venv/bin/activate
```

### Instalar Dependencias

Con el entorno virtual activado, instala los paquetes necesarios ejecutando:

```bash
pip install -r requirements.txt
```

## Paso 5: Iniciar el Servidor FastAPI

Con tu configuración de Visual Studio Code y las variables de entorno establecidas, inicia el servidor FastAPI:

1. Abre Visual Studio Code.
2. Abre la paleta de comandos (Ctrl+Shift+P).
3. Selecciona `Debug: Start Debugging`.

### Iniciar Manualmente

También puedes iniciar el servidor manualmente usando uvicorn:

```bash
uvicorn main:app --host 0.0.0.0 --port 8080
```

Esto iniciará el servidor en `http://0.0.0.0:8080`.

## Paso 6: Configurar Webhooks

### Configuración de Webhook de Twilio

Configura Twilio para enviar mensajes a tu servicio cuando un evento nuevo ocurre.

1. Ve al [Twilio Console](https://www.twilio.com/console).
2. En la sección de servicios de mensajería, configura la URL del webhook para apuntar a tu servidor FastAPI, por ejemplo: `http://<your-server-url>/webhook/twilio`.

### Configuración de Webhook de JIRA

Configura JIRA para enviar eventos de issue y comment creation a tu servicio de FastAPI. 

1. En tu cuenta JIRA, ve a `System > Webhooks`.
2. Crea un nuevo webhook con las URLs adecuadas:
    - Issue creation: `http://<your-server-url>/webhook/jira/issue`
    - Comment creation: `http://<your-server-url>/webhook/jira/comment`

## Paso 7: Pruebas Locales con ngrok

Para pruebas locales, es útil usar ngrok para exponer tu servidor local a Internet.

### Instalar ngrok

Descarga e instala ngrok desde [aquí](https://ngrok.com/download).

### Usar ngrok

Ejecuta ngrok para exponer tu servidor local:

```bash
ngrok http 8080
```

Esto te proporcionará una URL pública que puedes usar para configurar los webhooks de Twilio y JIRA durante tus pruebas locales.

## Resumen

Siguiendo estos pasos, podrás poner en marcha un entorno de desarrollo para probar el agente configurado para manejar eventos y procesos de soporte técnico. Este entorno incluye la configuración de Visual Studio Code, una base de datos MongoDB, variables de entorno y el servidor FastAPI, además de la configuración de webhooks para Twilio y JIRA.
